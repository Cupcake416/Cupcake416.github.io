魔方的表示

我们要如何表示一个魔方呢？一个最直观的想法是记录六个面54个色块的颜色，但实际上这是一种十分低效的方法，因为有些色块的相邻关系是确定的。事实上我们复原魔方时也是一层一层的复原，而不是孤立的关注单个面的颜色，受到这个的启发，我们可以用各个棱块和角块的位置和方向来表示一个魔方。

角块方向的表示

角方块有三个面，其中必有一个面为顶面或底面，这里我们取蓝色为底面，绿色为顶面，那么我们就可以这么规定一个角块的方向：先把一个角块像上图一样对准我们，并以蓝色或绿色为顶面，上图中三种角块的方向分别为0，1，2。

棱块方向的表示

棱块方向的表示会比角块更复杂一点，因为棱块只有两个面，所以只选取一对面作为基准面是不够的。我们选择白黄作为第二基准面，也就是说，当棱块含有蓝或绿时以蓝绿为基准，否则以白黄为基准。

Kociemba 算法

首先我们要找到魔方的一种中间状态：仅仅通过 U、D、L2、R2、F2、B2 这几种旋转就可以将魔方复原的状态。也就是不允许 90∘ 地旋转魔方的前后左右这四个面。这种状态有什么特点呢？
我们刚刚限定的旋转方法，是不会改变棱块和角块的方向的。那也就是说，在这种中间状态下，魔方的棱块和角块的方向应该都是 0，这是因为目标状态的魔方如此。除此之外，魔方中间层的四个棱块必然也在中间层。
换句话说，我们要找的中间状态要满足下面两个条件：
1.魔方的棱块和角块的方向都是 0
2.魔方中间层的四个棱块都在中间层

到这里，这个算法就自然的分为了两个部分：
1.把魔方旋转到中间状态
2.把魔方从中间状态复原

如何将魔方旋转到中间状态呢？我们使用 IDA* 来进行搜索。对于启发函数，分别针对角块的方向（一共 38 种可能）、棱块的方向（一共 28 种可能）、中间层棱块的位置及方向（一共 A412×24 种可能）计算复原它们所需要的最少步数。

然后是第二步，经过计算我们发现这一步的状态只有 8!×8!×4!=3.9×10^10 种可能，我们用 IDA* 来进行搜索，当我们设计启发函数的时候分别针对角块（一共 8! 种可能）、顶层和底层的棱块（一共 8! 种可能）、中间层的棱块（一共 4! 种可能）计算复原它们所需要的最少步数（这可以直接进行搜索），之后的启发函数就是这三者的最大值。

有些情况下，我们第一步就还原了魔方，那么这个结果是最优的。
